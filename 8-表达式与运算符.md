# coflow表达式与运算符设计

coflow表达式语法应清晰，脚本化，容易解析
逻辑运算使用单词形式
不使用C风格的`&&`，`||`，`!`

# 字面量
支持基础字面量

```coflow
123
3.14
true
false
null
"hello"
```

# 容器字面量
支持数组，字典，对象字面量

```coflow
var nums = [1, 2, 3]

var scores = {
  "alice": 10,
  "bob": 20,
}

weapon = {
  id: "sword",
  damage: 10,
}
```

# 访问表达式
支持字段访问，索引访问，可选链

```coflow
player.hp
items[0]
scores["alice"]
player?.weapon?.name
```

# 调用表达式
支持位置参数调用

```coflow
spawn("slime", 10, 5)
```

支持命名参数调用

```coflow
spawn(id: "slime", x: 10, y: 5)
```

# 算术运算
支持常见算术运算

```coflow
a + b
a - b
a * b
a / b
a % b
```

# 字符串拼接
字符串拼接使用`+`

```coflow
var text = "hello " + name
```

# 比较运算
支持比较运算

```coflow
a == b
a != b
a < b
a <= b
a > b
a >= b
```

# 逻辑运算
逻辑运算使用and，or，not

```coflow
if alive and hp > 0 {
  update()
}

if dead or hp <= 0 {
  respawn()
}

if not visible {
  show()
}
```

# is类型判断与收窄
is用于类型判断
is可以在if分支中进行类型收窄

```coflow
fn print_id(id: int | string) {
  if id is string {
    print(id.length)
  } else {
    print(id + 1)
  }
}
```

第一版只支持简单形式的类型收窄

```coflow
if value is Type {
  // value narrowed to Type
}
```

暂不支持复杂布尔表达式中的高级收窄

# 空值合并
空值合并使用`??`

```coflow
var name = player.name ?? "unknown"
var damage = weapon.damage ?? 0
```

只有左侧为null或缺失时才使用右侧

# 赋值运算
支持普通赋值和复合赋值

```coflow
a = b
hp += 10
hp -= 5
count *= 2
speed /= 2
count %= 2
```

# 运算符优先级
从高到低：

1. 调用，字段访问，索引访问，可选链
2. 一元`not`，一元`-`
3. `*`，`/`，`%`
4. `+`，`-`
5. `<`，`<=`，`>`，`>=`
6. `is`
7. `==`，`!=`
8. `and`
9. `or`
10. `??`
11. 赋值，复合赋值

# 暂不支持
1. `&&`
2. `||`
3. `!`
4. 三元表达式
5. 位运算
6. 复杂模式匹配
