# coflow类型系统设计

coflow采用简单渐进式类型系统
动态语言为底，类型系统为增强
类型主要服务于配置校验，宿主互操作，编辑器补全和错误诊断

# 设计原则
1. 类型标注可选
2. 类型默认可擦除
3. 不追求复杂类型表达能力
4. 支持简单联合类型
5. 不支持字面量联合
6. 支持is类型判断与基础收窄
7. 不支持复杂类型级编程
8. 运行时保留动态语言特性
9. 运行时支持鸭子类型

# 基础类型
1. int
2. float
3. bool
4. string
5. null
6. any

```coflow
var hp: int = 100
var speed: float = 3.5
var name: string = "hero"
var alive: bool = true
var target: any = get_target()
```

# 类型标注可选
未标注类型时保持动态语言体验

```coflow
var hp = 100
var name = "hero"
var data = load_data()
```

需要工程约束时再添加类型

```coflow
var hp: int = 100
var name: string = "hero"
```

# any类型
any表示任意动态值
主要用于宿主互操作，动态数据，迁移代码和无法静态确定类型的场景

```coflow
var value: any = host.get("player")
value.move(1, 0)
```

# null类型
null用于表示显式空值
需要允许空值时使用联合类型

```coflow
var name: string | null = null
var target: Entity | null = find_target()
```

null表示显式空值
缺失字段不等同于null

# 数组类型
数组表示有序列表

```coflow
var damages: [int] = [10, 20, 30]
var names: [string] = ["a", "b", "c"]
var mixed: [any] = [1, "hello", true]
```

# 字典类型
字典表示键值映射

```coflow
var scores: [string: int] = {
  "alice": 10,
  "bob": 20,
}
```

# class类型
class是主要结构声明方式
用于定义对象结构，方法集合和配置实例类型

```coflow
class Weapon {
  id: string
  name: string
  damage: int
  cooldown: float
}
```

配置是代码的一部分
顶层`name = value`定义自动视为配置
顶层配置可以选择添加类型标注
顶层`name: Type = value`表示带类型标注的配置定义
这些语法在顶层表示配置定义，不表示普通局部变量
顶层`var`定义普通顶层变量，不自动视为配置
顶层`var`默认公开，文件内私有变量使用`local var`
配置项默认对宿主可见，不需要额外export
配置项必须是常量，或者编译器可以确认为常量
配置项必须在编译期确定
函数值是合法的常量值
配置项不应依赖顶层var的运行时计算结果

```coflow
sword_untyped = {
  id: "sword"
  name: "Iron Sword"
  damage: 10
  cooldown: 0.8
}

sword: Weapon = {
  id: "sword"
  name: "Iron Sword"
  damage: 10
  cooldown: 0.8
}
```

# 配置常量表达式
配置项允许使用常量表达式

允许：
1. 字面量
2. 数组，字典，对象字面量
3. enum值
4. 函数值
5. 简单算术
6. 字符串拼接
7. 同文件其他配置项引用
8. import进来的其他文件配置常量
9. 对象和数组展开
10. 编译期可确认的切片
11. 编译期可确认的if表达式和match表达式
12. 编译器可确认的常量表达式

不允许：
1. 依赖顶层var
2. 在常量求值阶段调用普通函数
3. 调用宿主API
4. 访问IO，随机数，时间

```coflow
base_weapon: Weapon = {
  id: "base_sword"
  name: "Base Sword"
  damage: 10
  cooldown: 0.8
}

strong_weapon: Weapon = {
  id: "strong_sword"
  name: "Strong " + base_weapon.name
  damage: base_weapon.damage + 5
  cooldown: base_weapon.cooldown
}
```

常量表达式可以使用编译期可确认的展开，切片，if表达式和match表达式

```coflow
default_items = [
  "coin",
  "potion",
]

starter_items = [
  ...default_items,
  "sword",
]

visible_items = starter_items[0..2]

drop_text = if starter_items.length > 0 {
  "has_drop"
} else {
  "empty"
}

rarity_weight = match Rarity.rare {
  Rarity.common => 10,
  Rarity.rare => 30,
  Rarity.epic => 100,
}
```

配置项可以引用import进来的其他文件配置常量

```coflow
import weapons

starter_weapon: Weapon = {
  id: "starter_sword"
  name: weapons.base_weapon.name
  damage: weapons.base_weapon.damage
  cooldown: weapons.base_weapon.cooldown
}
```

函数值可以作为配置常量值

```coflow
class Skill {
  id: string
  apply: fn(caster: Entity, target: Entity) -> void
}

fireball: Skill = {
  id: "fireball"
  apply: fn(caster: Entity, target: Entity) -> void {
    target.hp -= 10
  }
}
```

函数值本身是常量
但配置常量求值阶段不调用普通函数

# 字段默认值
class可以声明字段默认值

```coflow
class Enemy {
  id: string
  hp: int = 100
  speed: float = 1.0
}

slime: Enemy = {
  id: "slime"
}
```

# 函数类型
函数参数和返回值可以标注类型

```coflow
fn add(a: int, b: int) -> int {
  return a + b
}
```

函数类型可以用于字段和宿主API

```coflow
class Skill {
  id: string
  apply: fn(caster: Entity, target: Entity) -> void
}
```

# enum类型
enum用于表达有限选项
不使用字面量联合表达有限选项

```coflow
enum Rarity {
  common
  rare
  epic
}

class Item {
  id: string
  rarity: Rarity
}

gem: Item = {
  id: "gem"
  rarity: Rarity.rare
}
```

# 联合类型
支持简单联合类型
联合类型用于编译器检查，配置校验和工具链提示

```coflow
var id: int | string
var target: Entity | null
```

```coflow
class ItemRef {
  id: int | string
}

item_a: ItemRef = {
  id: 1001
}

item_b: ItemRef = {
  id: "sword"
}
```

validate可以使用联合类型作为返回值

```coflow
class Weapon {
  damage: int

  fn validate(self) -> null | string | [string] {
    if self.damage <= 0 {
      return "damage must be greater than 0"
    }

    return null
  }
}
```

不支持字面量联合

```coflow
var mode: "easy" | "hard" // 不支持
```

有限选项使用enum

# 运行时鸭子类型
运行时支持鸭子类型
只要对象在运行时拥有对应字段或方法，就允许动态访问

```coflow
fn update(obj: any) {
  obj.move(1, 0)
}
```

如果obj运行时没有move方法，则运行时报错

# 配置类型校验
带类型标注的顶层配置定义可以按class类型校验
未标注类型的配置仍然是合法配置
未标注配置按推断类型或动态配置处理

```coflow
class DropConfig {
  item_id: string
  weight: int
}

slime_drop: DropConfig = {
  item_id: "coin"
  weight: 100
}

raw_drop = {
  item_id: "coin"
  weight: 100
}
```

类型错误应在加载或检查阶段诊断

```coflow
bad_drop: DropConfig = {
  item_id: "coin"
  weight: "high"
}
```

# 宿主边界类型
宿主函数可以导出类型签名
用于参数转换，返回值检查，错误诊断和编辑器补全

```coflow
extern fn spawn(id: string, x: float, y: float) -> Entity

var enemy = spawn("slime", 10.0, 5.0)
```
