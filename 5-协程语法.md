# coflow协程语法设计

coflow支持轻量协作式协程
协程用于游戏脚本，工具脚本，流程控制和宿主调度
第一版采用stackless coroutine
项目计划使用基于寄存器的VM实现

# 设计原则
1. 语法显式
2. 协作式调度
3. 不做抢占式线程
4. 宿主负责调度器
5. 语言只提供协程对象，yield，resume
6. co fn显式标记协程入口
7. yield只允许直接出现在co fn函数体内
8. 普通fn和宿主回调不能yield

# 协程函数
使用co fn声明协程函数
co fn是协程入口函数
co fn必须至少包含一个yield

```coflow
co fn blink(entity: Entity) {
  while true {
    entity.visible = false
    yield
    entity.visible = true
    yield
  }
}
```

调用co fn不会立即执行完
调用结果是coroutine对象

```coflow
var routine = blink(player)
routine.resume()
routine.resume()
```

# 匿名协程函数
支持匿名co fn

```coflow
var task = co fn() {
  yield
}
```

协程函数可以作为配置常量值

```coflow
cutscene = {
  run: co fn() {
    yield wait.seconds(1.0)
  }
}
```

# yield语句
yield只能直接出现在co fn函数体内
普通fn，匿名普通fn，宿主回调和validate函数中都不能使用yield
yield是语句，不是表达式

```coflow
co fn task() {
  yield
  yield 1
  yield "wait"
}
```

无值yield等价于yield null

co fn必须至少包含一个yield

```coflow
co fn invalid() {
  return 1
}
```

上例错误，因为co fn没有yield

# resume结果
resume返回结果对象

```coflow
{
  done: bool
  value: any
}
```

示例：

```coflow
co fn counter() {
  yield 1
  yield 2
}

var c = counter()

c.resume() // { done: false, value: 1 }
c.resume() // { done: false, value: 2 }
c.resume() // { done: true, value: null }
```

# return语句
co fn中的return表示协程最终结果

```coflow
co fn task() -> int {
  yield 1
  return 10
}
```

最后一次resume返回：

```coflow
{ done: true, value: 10 }
```

# yield协程自动委托
yield可以接收另一个coroutine对象
当yield的值是coroutine时，当前协程自动委托执行子协程
子协程完成后，父协程继续执行
这是显式委托，不表示普通函数调用栈可以被暂停

```coflow
co fn child() {
  yield 1
  yield 2
}

co fn parent() {
  yield child()
  yield 3
}
```

执行结果：

```coflow
var c = parent()

c.resume() // { done: false, value: 1 }
c.resume() // { done: false, value: 2 }
c.resume() // { done: false, value: 3 }
c.resume() // { done: true, value: null }
```

# 普通函数边界
co fn可以调用普通函数
普通函数必须在返回后，co fn才能继续执行或yield
普通函数内部不能yield

```coflow
fn log_start() {
  print("start")
}

fn prepare_wait_message() -> string {
  return "click"
}

co fn task() {
  log_start()
  yield wait.event(prepare_wait_message())
}
```

普通函数中使用yield是错误

```coflow
fn helper() {
  yield // 错误
}
```

# VM实现模型
在基于寄存器的VM中，co fn可以编译为普通字节码加YIELD指令
coroutine对象保存函数引用，寄存器数组，pc和状态

```text
Coroutine {
  function
  registers
  pc
  state: new | running | suspended | done | dead
}
```

resume从保存的pc和寄存器继续执行
执行到YIELD时保存下一条指令位置并返回结果对象
不需要保存普通fn调用栈

# 已完成协程
已完成协程再次resume返回done结果

```coflow
{ done: true, value: null }
```

# 错误处理
协程内部发生错误时
resume传播运行时错误
协程进入dead状态

```coflow
co fn task() {
  throw "failed"
}

var c = task()

try {
  c.resume()
} catch err {
  print(err.message)
}
```

# 宿主调度
语言核心不内置等待秒数，等待帧，等待事件
这些能力由宿主或标准库提供

```coflow
co fn cutscene() {
  yield wait.seconds(1.0)
  yield wait.event("click")
  yield wait.frame()
}
```

# 暂不支持
1. 抢占式线程
2. async/await
3. yield作为表达式
4. resume传值回yield
5. 普通fn中yield
6. yield穿过宿主回调
