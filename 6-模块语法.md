# coflow模块语法设计

coflow模块系统以文件为模块
模块语法应简单，明确，适合配置和脚本共同使用

# 基本规则
1. 一个文件是一个模块
2. 使用import导入其他模块
3. 使用模块名访问导入模块的成员
4. 顶层声明默认公开
5. 使用local声明文件内私有成员
6. 配置项默认公开
7. 自定义类型默认公开
8. 顶层var，fn，co fn默认公开

# import
使用import导入模块

```coflow
import weapons

starter = {
  id: "starter_sword"
  damage: weapons.sword.damage
}
```

支持import别名
别名只改变当前文件中的模块访问名

```coflow
import weapons as w

starter = {
  id: "starter_sword"
  damage: w.sword.damage
}
```

import别名必须是简单标识符
同一文件内别名不能与已有顶层声明或其他import名冲突

# 配置项公开
顶层配置项默认公开
不需要export或pub

```coflow
sword = {
  id: "sword"
  damage: 10
}
```

其他模块可以访问：

```coflow
import weapons

var damage = weapons.sword.damage
```

# 自定义类型公开
自定义类型默认公开
包括class和enum

```coflow
class Weapon {
  id: string
  damage: int
}

enum Rarity {
  common
  rare
  epic
}
```

其他模块可以访问：

```coflow
import weapons

starter: weapons.Weapon = {
  id: "starter_sword"
  damage: 10
}
```

# local声明
使用local声明文件内私有成员
local成员只能在当前文件内使用

```coflow
local class InternalWeaponData {
  base_damage: int
}

local enum InternalKind {
  normal
  special
}

local var base_damage = 10

local fn helper() -> int {
  return base_damage
}
```

# 配置不能使用local类型
配置项默认公开
因此配置项不能使用local类型

不允许：

```coflow
local class InternalWeapon {
  id: string
}

weapon: InternalWeapon = {
  id: "sword"
}
```

# 顶层var
顶层var默认公开
如果只作为文件内辅助变量，应使用local var

```coflow
var base_damage = 10
local var internal_damage = 5
```

# 顶层函数
顶层fn和co fn默认公开
如果只作为文件内辅助函数，应使用local fn或local co fn

```coflow
fn helper() -> int {
  return 10
}

co fn task() {
  yield
}

local fn internal_helper() -> int {
  return 5
}

local co fn internal_task() {
  yield wait.seconds(1.0)
}
```

# 暂不支持
1. export关键字
2. from import
3. 通配符import
4. 循环import
