# 字节码指令集设计

本文档已迁移至 [docs/design/bytecode.md](../design/bytecode.md)。

实现步骤见下方。

## 实现顺序

1. 定义 `OpCode` 枚举与 `Proto` 结构（`bytecode.rs`）
2. 实现指令编解码（packing/unpacking 32 位指令）
3. 实现常量池构建器（主池 + consts_k + field_names）
4. 实现 codegen 骨架（HIR → Proto），先支持表达式与基础语句
5. 实现寄存器分配（Lua 5.x 风格的虚拟栈）
6. 实现控制流 codegen（if / while / loop / for in / break / continue），用 JFALSE / JTRUE / JMP_LOOP
7. 实现闭包 codegen（CLOSURE / CLOSE / upvalue 链）
8. 实现 iter fn codegen（YIELD / ITERLOOP）
9. 实现 try/catch codegen
10. 实现 TypeGuard 插入与 TypeDescriptor 序列化
11. 实现 STACKCHECK 注入
12. 实现 CALLKW + NamedArgList
13. 实现 INVOKE（含 EXTRAARG）
14. 实现 SETLIST、CONCATN、LEN、ISNULL、BRNULL/BRNNULL、NULLCOALESCE
15. 实现 K 版本算术与比较（ADDK / SUBK / MULK / EQK / LTK / LEK）
16. 反汇编器（调试用）
17. 第二阶段：加分化指令、Inline Cache 槽、LOADKX 兜底

## 测试清单

测试分为编码、codegen、反汇编三类。

### 指令编解码

1. ABC / ABx / AsBx 三种格式正确编码与解码
2. sBx 负数偏移正确处理
3. EXTRAARG 紧跟前一条指令，24 位扩展索引正确解码
4. OpCode 256 上限不溢出

### codegen

1. 简单表达式 `1 + 2` → LOADINT + LOADINT + ADD + RETURN
2. 常量内联：`x + 1` → ADDK
3. 局部变量 `var x = 1; x + 1` → LOADINT + ADDK
4. if 条件跳转用 JFALSE 而非 TEST + JMP
5. while 循环回边用 JMP_LOOP
6. for in range 翻 FORPREP/FORLOOP
7. for in 数组翻 ITER + ITERLOOP
8. 闭包捕获：CLOSURE 的 upvalue_descs 与 HIR 一致
9. 多级闭包：内层 Upvalue(idx) 链路正确
10. 函数调用：参数寄存器连续放置
11. 具名参数：CALLKW 的 NamedArgList 正确
12. 方法调用：INVOKE + EXTRAARG 正确
13. try/catch：TRY 偏移指向 catch 块开头
14. yield：iter fn 的 YIELD 正确
15. ?. 翻 GETFIELDOPT
16. ?? 翻 NULLCOALESCE
17. and/or 翻 TESTSET 短路链
18. 链式比较翻 AndChain → JFALSE 链
19. TypeGuard 翻 TYPECHECK + TypeDescriptor
20. STACKCHECK 在函数入口和循环 body 入口出现
21. 大数组字面量翻 NEWARRAY + SETLIST
22. 多元字符串拼接翻 CONCATN
23. `.length` 翻 LEN
24. `x == null` 翻 BRNULL（条件分支语境）或 ISNULL（值语境）
25. `x != null` 翻 BRNNULL

### 反汇编

1. 每条指令能反汇编成可读形式（用于调试）
2. 反汇编输出与源 HIR span 对应（line_info 正确）
3. 主常量池条目能可视化
4. consts_k 与 field_names 子表能可视化
