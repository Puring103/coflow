# 设计理念

Coflow 不是又一个"更好用的导表脚本"。它把游戏配置的生产链路当作**一整套工程问题**来解决：从字段建模、多源采集、业务校验，到运行时交付、变体展开、AI 协作。

要理解 Coflow 的取舍，最快的方式是先看清楚传统导表工作流的痛点，再看 Coflow 是如何回应的。

## 传统导表工作流的痛点

### A. 项目/工程层：每个项目一套方案

- 每上一个新项目，就要重写一遍导表脚本、错误码、目录约定
- 换团队/换引擎（Unity ↔ Unreal ↔ 自研），导表逻辑几乎没法复用
- Excel 表结构改一次，导表脚本、加载器、DTO 三处都要跟着改
- 配置本身没有版本管理概念，回滚、diff、review 都靠"人肉记忆"

### B. 数据模型层：类型不明确

- 单元格里塞 JSON、逗号分隔数组、`|` 分隔的 key:value —— 全靠"团队约定"
- 引用另一张表用字符串 ID，拼错要到运行时才炸
- 多态字段（同一列出现不同种类的对象）几乎无法建模
- 默认值和业务规则散落在导表脚本、loader、策划脑子里，schema 本身不完整

### C. 数据质量层：数据不可信

- 主键重复没人拦
- 字段被删了，但引用它的表还在
- 半成品的表格能通过导表进入包体
- "运行时崩溃"是发现配置错误最常见的方式

### D. 诊断/排错层：报错说不清

- 导表脚本报 `KeyError: 'atk_bonus'` —— 不知道哪张表、哪一行
- 加载失败只有一句 `Invalid config`
- CI 挂了要读几百行栈才能定位
- 找错基本靠 grep Excel，嵌套字段连 grep 都不好使

### E. 协作/写回层：改了不敢提

- 策划想改 Excel，程序说"要走导表规范"
- 多种本地格式的结构不同，容易形成多套加载和校验逻辑
- 想让 AI 或自动化工具写回配置，只能整份文件覆盖，没有 patch 语义
- Excel 二进制 diff = 一坨乱码，review 时几乎看不出改了什么

### F. 交付层：手写 loader 与 DTO 漂移

- 每加一个字段：schema 改一处、DTO 改一处、导表脚本改一处、loader 改一处 —— 四处对齐
- 手写 C# 类经常滞后于 Excel 结构，运行时才发现漂移
- 想切换到 MessagePack / FlatBuffers / 自研二进制，要再写一套导出
- 本地化通常单开一套流程，和主配置解耦

### G. 变体层：本地化、平台、渠道各一套

- 语言/平台/渠道/区服的差异散落在多张表或多个字段
- "美服要禁掉这个 buff"，常见做法是加一列 `us_only`，schema 本身没有这个概念
- 要出 iOS 特供、渠道特供 —— 要么复制表，要么加 if
- 变体的 QA 覆盖不到全排列

### H. AI 协作层：AI 不友好

- AI 拿到 Excel 二进制根本读不懂
- AI 想生成配置，只能靠"看示例猜格式"
- AI 想修一个字段，只能整行/整表重写
- 没有稳定的 CLI 让 AI 消费诊断、提交 patch、拿到反馈继续迭代
- 把 schema 说明塞进 prompt 费 token，还不一定准

## Coflow 的回应

Coflow 不是"分别解决"上面每一条，而是从一个更基础的选择开始：**把配置当作有类型的数据资产**，并围绕这个选择重建整套工作流。

| 痛点档 | Coflow 的回应 |
|---|---|
| A. 每项目一套 | CLI + engine + provider + codegen 通用工程链路，跨项目零重写 |
| B. 类型不明确 | CFT schema：类型、枚举、默认值、引用、多态、check 都是一等公民 |
| C. 数据不可信 | 引用解析、check、去重、跨表约束都在同一个 runtime model 上跑 |
| D. 报错说不清 | 结构化诊断：`code · stage · file · sheet · cell · record · field path` |
| E. 改了不敢提 | Provider writer 支持 patch、写回原始数据源（Excel / CSV / CFD），改动可复盘 |
| F. 手写 loader | 一次构建同时产出运行时数据（JSON / MessagePack）和加载代码（C#） |
| G. 本地化割裂 | Dimension 描述任意维度的变体：语言、平台、渠道、区服，同一份 schema 展开 |
| H. AI 不友好 | 稳定 CLI 入口 + 结构化 patch + 结构化诊断，AI agent 可以闭环消费 |

## 一句话总结

> **Coflow 让配置数据像代码一样可信** —— 类型驱动、多源接入、结构化诊断、可写回、可交付、AI 可维护。
