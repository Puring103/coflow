# coflow语言整体设计思想
coflow是一种面向嵌入式场景的现代动态脚本语言，目标是替代lua，同时将配置作为代码的一部分

# 核心目标
1. 现代化的lua替代品
2. 配置友好的脚本语言
3. 顶层配置定义自动视为配置
4. 统一脚本逻辑，配置数据，配置校验
5. 轻量级，可嵌入
6. 与宿主语言便捷互操作

# 目标领域
1. 嵌入式脚本语言
2. 带逻辑，自校验的复杂配置
3. 游戏，工具，编辑器扩展脚本
4. UGC脚本与数据描述

# 核心特点
1. 现代化语法
2. 动态灵活
3. 渐进式类型系统
4. 配置是第一公民
5. 清晰的数据模型
6. 清晰的模块系统
7. 清晰的空值与缺失值语义
8. 轻量级，可嵌入
9. 较高的性能
10. 与宿主语言的便捷互操作
11. 良好的工具链支持

# 相比lua需要改进的问题
1. 语法老旧
2. 隐式全局配置诊断与管理
3. table语义过度统一
4. nil语义过载
5. 缺乏类型系统
6. 模块系统不够现代
7. 错误处理不够现代
8. 大型项目工具链支持弱

# 配置能力
1. 配置是代码的一部分
2. 顶层`name = value`定义自动视为配置
3. 顶层配置可以选择添加类型标注
4. 配置本质上是顶层常量定义，不是变量
5. 顶层`var`定义文件内辅助变量
6. 配置项默认对宿主可见
7. 不需要额外export配置项
8. 声明结构化配置数据
9. 支持默认值
10. 支持常量表达式
11. 支持派生字段
12. 支持字段类型
13. 支持结构约束
14. 支持跨字段校验
15. 支持配置组合，拆分，复用
16. 支持宿主统一加载，校验，消费
17. 配置必须在编译期确定

# 配置常量表达式
1. 支持字面量
2. 支持数组，字典，对象字面量
3. 支持enum值
4. 支持函数值
5. 支持简单算术
6. 支持字符串拼接
7. 支持同文件其他配置项引用
8. 支持import其他文件配置常量
9. 支持编译器可确认的常量表达式
10. 不依赖顶层var的运行时结果
11. 不在常量求值阶段调用普通函数
12. 不调用宿主API
13. 不访问IO，随机数，时间
14. 配置常量表达式必须在编译期求值或确认

# 支持特性
1. 动态脚本执行
2. 渐进式类型标注
3. 配置声明与校验
4. 模块导入
5. 宿主函数绑定
6. 宿主类型绑定
7. 错误诊断
8. 编辑器补全与静态分析
9. 现代控制流语法
10. 现代基础语法
11. 函数是一等值
12. 可选链，空值合并，命名参数，字符串插值
13. 轻量协作式协程
14. 文件级模块系统
15. class结构声明与配置校验

# 类型系统
1. 简单渐进式类型系统
2. 动态语言为底，类型系统为增强
3. 类型标注可选，默认可擦除
4. 运行时支持鸭子类型
5. 支持简单联合类型
6. 不支持字面量联合
7. 不支持T?可空语法糖
8. 顶层配置定义可按类型校验
9. 宿主边界可按类型转换和诊断
10. 详细设计见《1-类型系统.md》

# 控制流语法
1. 使用花括号代码块
2. 支持if，else if，else
3. 支持while
4. 支持for-in统一循环
5. 支持break，continue，return
6. 详细设计见《2-控制流语法.md》

# 基础语法
1. 支持单行注释和块注释
2. 局部变量使用var
3. var为块级作用域
4. 顶层var为文件内辅助变量
5. 支持隐式全局配置
6. 支持普通赋值，字段赋值，索引赋值
7. 支持复合赋值
8. 不支持lua式多重赋值
9. 不支持++和--
10. 支持可选链和空值合并
11. 支持尾随逗号
12. 支持字符串插值
13. 详细设计见《3-基础语法.md》

# 函数语法
1. 函数是一等值
2. 支持普通函数和匿名函数
3. 参数类型和返回类型可选
4. 支持命名参数调用
5. 第一版暂不支持闭包捕获
6. 第一版不支持lua式多返回值
7. 详细设计见《4-函数语法.md》

# 协程语法
1. 支持stackful coroutine
2. 调用co fn返回coroutine对象
3. 支持yield语句
4. resume返回done和value
5. yield另一个协程时自动委托执行
6. 调度器由宿主提供
7. co fn显式标记协程入口
8. yield可以暂停整个协程调用栈
9. 第一版不支持async/await
10. 详细设计见《5-协程语法.md》

# 模块语法
1. 一个文件是一个模块
2. 使用import导入模块
3. 顶层声明默认公开
4. 使用local声明文件内私有成员
5. 配置项默认公开
6. 自定义类型默认公开
7. 顶层var，fn，co fn默认公开
8. 配置不能使用local类型
9. 不使用pub和export
10. 详细设计见《6-模块语法.md》

# class与配置校验
1. class声明对象结构和方法
2. 配置必须在编译期确定
3. 带类型配置按class结构校验
4. 缺失必填字段报错
5. 带类型配置不允许额外字段
6. class可定义validate自定义校验
7. 普通对象可变
8. 配置对象默认只读
9. 第一版不支持继承
10. 详细设计见《7-class与配置校验.md》
