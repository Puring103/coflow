主数据流

`coflow check`、`coflow export` 和 `coflow codegen` 都复用前半段 schema 编译流程。

1

打开项目

读取 `coflow.yaml`，解析 schema、Excel sources 和输出目录。

2

编译 CFT

发现 `.cft` 文件，生成 `CftContainer`，收集稳定诊断。

3

加载 Excel

按 sheet 和列映射读取 `.xlsx`，把单元格解析为输入值。

4

构建模型

生成 `CfdDataModel`，处理默认值、继承、表和引用。

5

运行检查

`coflow-checker` 执行 schema 中的 `check` 规则。

6

导出数据

通过 exporter-core 遍历模型，输出 JSON 或 MessagePack。

7

生成代码

按编译后的 schema 生成 C# 类型和可信数据加载器。

模块分层

底层 crate 提供稳定模型和语义，上层 crate 只做编排、格式适配或编辑器集成。

语言与模型层

  • coflow-cftCFT 词法、语法、schema 编译、类型检查和诊断。
  • coflow-data-model来源无关的数据模型、表、记录、值和引用解析。
  • coflow-cell-value把 Excel 单元格文本按 schema 解析成输入值。

管线与校验层

  • coflow-project解析项目配置、发现 schema 文件、编译项目 schema。
  • coflow-loader-excel读取 `.xlsx`，映射 sheet/列，构建并校验模型。
  • coflow-checker执行 CFT `check` 块，返回数据诊断。

输出与工具层

  • coflow-exporter-core共享的 schema-aware 导出遍历。
  • coflow-exporter-json / messagepack具体数据格式编码器。
  • coflow-codegen-csharpC# 运行时模型和加载器生成。
  • coflow-cft-lspCFT 语言服务，供 VS Code 扩展调用。

CLI 命令入口

coflow init

创建最小项目结构、schema 目录、data 目录和默认 `coflow.yaml`。

coflow cft check

只编译项目 schema，输出 human-readable 或 JSON 诊断。

coflow cft lsp

启动 CFT language server，服务 VS Code 扩展。

coflow check

完整运行 schema 编译、Excel 加载、模型构建和 check 规则。

coflow export / codegen

导出 JSON/MessagePack 数据，或生成 C# runtime 代码。
