# coflow 字节码指令集设计

本文定义 coflow 字节码 VM 的指令集，作为 codegen 与 VM 实现的契约。

定位：

1. 寄存器式 VM（参考 Lua 5.x，但调整为 8 位 OpCode）
2. 32 位定长指令
3. 第一版基础指令集，第二版加入分化指令做性能优化
4. 服务于"动态类型 + 高性能脚本语言"目标

核心依据：

1. `docs/plan/语义分析与HIR实现计划.md`
2. `docs/0-core-language.md`
3. `docs/1-core-types.md`
4. `docs/3-core-runtime.md`

## 设计原则

1. **寄存器式**：每条指令显式指定操作数和目标寄存器，dispatch 次数比栈式少 30-50%
2. **8 位 OpCode**：256 上限，留有充足扩展空间
3. **常量内联（K 版本）**：高频算术与比较有专用 K 版本，操作数直接是常量池索引，省 LOADK
4. **Inline cache 预留**：高频字段访问指令带 cache 槽位（不增加指令数）
5. **分化优化策略**：
   - 第一阶段：基础指令集，先跑通管线
   - 第二阶段：算术 / 比较分化，加 IC，按 profile 决定优先级

## 指令格式

32 位定长，三种格式：

```
ABC  格式: [ OpCode:8 | A:8 | B:8 | C:8 ]
ABx  格式: [ OpCode:8 | A:8 | Bx:16    ]
AsBx 格式: [ OpCode:8 | A:8 | sBx:16   ]
```

字段含义：

| 字段 | 用途 |
| --- | --- |
| OpCode | 8 位指令操作码，上限 256 |
| A | 8 位主寄存器号（多数指令的目标寄存器） |
| B | 8 位寄存器号或字段名常量索引或常量池索引（依指令） |
| C | 8 位寄存器号或字段名常量索引或常量池索引（依指令） |
| Bx | 16 位无符号常量池/Proto/Global 索引 |
| sBx | 16 位有符号偏移（跳转专用） |

寻址容量：

- 寄存器：256 个/函数（实际单函数极少超过 200）
- 主常量池（16 位）：65536 个条目（字符串、数字、TypeDescriptor、NamedArgList、Proto 等）
- 字段名子表（8 位）：256 个/Proto，专用于 GETFIELD / SETFIELD / GETFIELDOPT / INVOKE
- 算术 K 版本的常量索引（8 位）：256 个/Proto，独立于主常量池，专用于 ADDK / LTK 等
- 全局变量：65536 个

## 完整指令清单（基础版 78 条）

### 数据移动（6 条）

| OpCode | 格式 | 语法 | 语义 |
| --- | --- | --- | --- |
| MOVE | ABC | `MOVE rA, rB` | `rA = rB` |
| LOADK | ABx | `LOADK rA, Kx` | `rA = consts[Kx]` |
| LOADKX | ABC | `LOADKX rA` | `rA = consts[紧跟的 EXTRAARG 提供的 24 位索引]` |
| LOADINT | AsBx | `LOADINT rA, sBx` | `rA = sBx`（小整数内联） |
| LOADBOOL | ABC | `LOADBOOL rA, b` | `rA = (b != 0)`，B=0/1 |
| LOADNIL | ABC | `LOADNIL rA, count` | `rA..rA+count = null` |

辅助：

| OpCode | 格式 | 语法 | 语义 |
| --- | --- | --- | --- |
| EXTRAARG | ABx | `EXTRAARG Ax` | 仅作为前一条指令的扩展参数；24 位索引由 A:8 + Bx:16 拼接 |

LOADK 主路径覆盖 65536 个常量。LOADKX + EXTRAARG 是兜底机制，第一版可不实现，超过时编译报错；未来按需启用。

### 全局与 Upvalue（4 条）

| OpCode | 格式 | 语法 | 语义 |

| SETGLOBAL | ABx | `SETGLOBAL rA, G_idx` | `globals[G_idx] = rA` |

### 对象 / 字典 / 数组（16 条）

| OpCode | 格式 | 语法 | 语义 |
| --- | --- | --- | --- |
| NEWOBJECT | ABx | `NEWOBJECT rA, class_id` | `rA = {}`，class_id=0 表示无 class |
| NEWDICT | ABx | `NEWDICT rA, hint` | `rA = {}`，hint 是预分配大小 |
| NEWARRAY | ABx | `NEWARRAY rA, hint` | `rA = []` |
| GETFIELD | ABC | `GETFIELD rA, rB, Kf` | `rA = rB.<field_names[Kf]>` |
| SETFIELD | ABC | `SETFIELD rA, Kf, rC` | `rA.<field_names[Kf]> = rC` |
| GETFIELDOPT | ABC | `GETFIELDOPT rA, rB, Kf` | `rA = rB?.<field_names[Kf]>`（rB 为 null 时 rA = null） |
| GETINDEX | ABC | `GETINDEX rA, rB, rC` | `rA = rB[rC]` |
| SETINDEX | ABC | `SETINDEX rA, rB, rC` | `rA[rB] = rC` |
| GETI | ABC | `GETI rA, rB, rC` | `rA = rB[rC]`，rC 必须是 int（数组特化） |
| SETI | ABC | `SETI rA, rB, rC` | `rA[rB] = rC`，rB 必须是 int |
| APPEND | ABC | `APPEND rA, rB` | `rA.push(rB)`（rA 必须是数组） |
| SETLIST | ABC | `SETLIST rA, B, C` | `rA[C..C+B-1] = rA+1..rA+B`（数组批量初始化） |
| LEN | ABC | `LEN rA, rB` | `rA = len(rB)`，支持数组/字典/字符串 |
| SPREAD | ABC | `SPREAD rA, rB` | 把 rB 的所有字段/元素复制到 rA |
| RANGE | ABC | `RANGE rA, rB, rC` | `rA = rB..rC`（不含尾） |

`SETLIST` 用于数组字面量优化：`[1, 2, ..., 100]` 一条指令完成 100 个元素的批量写入，配置场景关键。

`LEN` 处理 `.length` 之类的内置长度查询，省一次字段查找+IC 槽。

### 算术（9 条 + K 版本 3 条）

| OpCode | 格式 | 语法 | 语义 |
| --- | --- | --- | --- |
| ADD | ABC | `ADD rA, rB, rC` | `rA = rB + rC`（数字/字符串运行时分派） |
| SUB | ABC | `SUB rA, rB, rC` | `rA = rB - rC` |
| MUL | ABC | `MUL rA, rB, rC` | `rA = rB * rC` |
| DIV | ABC | `DIV rA, rB, rC` | `rA = rB / rC`（浮点除法） |
| IDIV | ABC | `IDIV rA, rB, rC` | `rA = rB // rC`（整数截断除法） |
| REM | ABC | `REM rA, rB, rC` | `rA = rB % rC` |
| POW | ABC | `POW rA, rB, rC` | `rA = rB ** rC` |
| UNM | ABC | `UNM rA, rB` | `rA = -rB`（一元负号） |
| CONCAT | ABC | `CONCAT rA, rB, rC` | `rA = rB + rC`（字符串拼接专用） |
| CONCATN | ABC | `CONCATN rA, rB, rC` | `rA = concat(rB..rC)`（多元字符串拼接，省中间分配） |

K 版本（右操作数是常量索引，省 LOADK）：

| OpCode | 格式 | 语法 | 语义 |
| --- | --- | --- | --- |
| ADDK | ABC | `ADDK rA, rB, K` | `rA = rB + consts_k[K]` |
| SUBK | ABC | `SUBK rA, rB, K` | `rA = rB - consts_k[K]` |
| MULK | ABC | `MULK rA, rB, K` | `rA = rB * consts_k[K]` |

注：

- `ADD` 与 `CONCAT` 分开是因为 codegen 在已知类型时可发出更快的 CONCAT；`ADD` 兼容运行时类型分派
- `CONCATN` 处理 `a + b + c + d` 这类多元拼接，rB..rC 是连续寄存器范围
- K 版本只覆盖 ADD/SUB/MUL，其它（DIV/IDIV/REM/POW/位运算）出现常量参与运算的频率低，不加 K 版本

### 比较与逻辑（6 条 + K 版本 3 条）

| OpCode | 格式 | 语法 | 语义 |
| --- | --- | --- | --- |
| EQ | ABC | `EQ k, rB, rC` | `if (rB == rC) != bool(k): pc++`，A=期望值 |
| LT | ABC | `LT k, rB, rC` | `if (rB < rC) != bool(k): pc++` |
| LE | ABC | `LE k, rB, rC` | `if (rB <= rC) != bool(k): pc++` |
| NOT | ABC | `NOT rA, rB` | `rA = !bool(rB)` |
| IN | ABC | `IN rA, rB, rC` | `rA = rB in rC` |
| ISNULL | ABC | `ISNULL rA, rB` | `rA = (rB == null)` |

K 版本：

| OpCode | 格式 | 语法 | 语义 |
| --- | --- | --- | --- |
| EQK | ABC | `EQK k, rB, K` | `if (rB == consts_k[K]) != bool(k): pc++` |
| LTK | ABC | `LTK k, rB, K` | `if (rB < consts_k[K]) != bool(k): pc++` |
| LEK | ABC | `LEK k, rB, K` | `if (rB <= consts_k[K]) != bool(k): pc++` |

约定：

- 比较指令的 A 字段是"期望值"（0 或 1）。实际比较结果与期望不一致时跳过下一条
- `!=` 用 `EQ 0, ...`；`>` 用 `LT k, rC, rB`（交换操作数）；`>=` 用 `LE k, rC, rB`
- `ISNULL` 是 `x == null` 的专用形式，省一次 LOADNIL + EQ

### 跳转（8 条）

| OpCode | 格式 | 语法 | 语义 |
| --- | --- | --- | --- |
| JMP | AsBx | `JMP sBx` | `pc += sBx` |
| JMP_LOOP | AsBx | `JMP_LOOP sBx` | `pc += sBx`，标记循环回边（未来 JIT 用，第一版同 JMP） |
| JFALSE | AsBx | `JFALSE rA, sBx` | `if !bool(rA): pc += sBx` |
| JTRUE | AsBx | `JTRUE rA, sBx` | `if bool(rA): pc += sBx` |
| BRNULL | AsBx | `BRNULL rA, sBx` | `if rA == null: pc += sBx` |
| BRNNULL | AsBx | `BRNNULL rA, sBx` | `if rA != null: pc += sBx` |
| TEST | ABC | `TEST rA, k` | `if bool(rA) != bool(k): pc++` |
| TESTSET | ABC | `TESTSET rA, rB, k` | `if bool(rB) == bool(k): rA = rB; else: pc++` |
| NULLCOALESCE | ABC | `NULLCOALESCE rA, rB, rC` | `rA = (rB == null ? rC : rB)` |

约定：

- `JFALSE` / `JTRUE` 是 if/while 主路径，比 TEST + JMP 少一条指令
- `BRNULL` / `BRNNULL` 是 `x == null` / `x != null` 的分支特化
- `TEST` / `TESTSET` 仍保留用于 and/or 短路链
- `NULLCOALESCE` 是 `??` 的专用指令；`??=` 在 codegen 展开为 NULLCOALESCE 后跟 SETLOCAL/MOVE
- `JMP_LOOP` 第一版语义同 JMP；codegen 在循环回边发 JMP_LOOP，未来加 JIT 时不需改字节码

### 数值 for 循环（2 条）

| OpCode | 格式 | 语法 | 语义 |
| --- | --- | --- | --- |
| FORPREP | AsBx | `FORPREP rA, sBx` | 初始化数值 for；`rA`=当前值，`rA+1`=终止值；如初始已超出范围跳 sBx |
| FORLOOP | AsBx | `FORLOOP rA, sBx` | 自增 `rA`；若仍 `< rA+1`，跳 sBx 继续循环 |

约定：

- 步长固定为 1，专用于 `for i in start..end` 优化路径
- inclusive (`..=`) 在 codegen 阶段转换为 `..`（end+1 处理；INT_MAX 边界限制由文档说明）
- 通用迭代走 ITER / ITERLOOP

### 函数调用（5 条）

| OpCode | 格式 | 语法 | 语义 |
| --- | --- | --- | --- |
| CALL | ABC | `CALL rA, argc, retc` | 调用 `rA`，参数从 `rA+1..rA+argc`，返回值写回 `rA..rA+retc` |
| CALLKW | ABC | `CALLKW rA, argc_total, kw_idx` | 含具名参数的调用；`kw_idx` 指向常量池中的 NamedArgList |
| INVOKE | ABC | `INVOKE rA, rB, Kf` | `rA = rB.<field_names[Kf]>(rA+1..rA+argc)`；argc 由后续 EXTRAARG 提供 |
| TAILCALL | ABC | `TAILCALL rA, argc` | 尾调用（复用当前帧） |
| RETURN | ABC | `RETURN rA, has_value` | 返回 `rA`（has_value=1）或返回 null（has_value=0） |

调用约定：

- 函数和参数占连续寄存器：`rA`=函数对象，`rA+1..rA+argc`=参数
- 被调函数看到的 `reg[0]` 就是第一个参数（参考 Lua 5.x 调用窗口）
- 返回值写回 `rA` 起始位置；retc 指定调用方期望的返回值数量
- coflow 不支持多返回值，retc 实际为 0 或 1
- CALLKW 的 NamedArgList 描述哪些位置是具名参数及对应名字，VM 按名字映射到 callee 的形参位置并填充默认值
- `INVOKE` 是"取字段+调用"的合并指令，配合 IC 优化字段查找；调用窗口为 `rA+1..rA+argc`，`rB` 不进入参数列表（它就是接收者本身，但 coflow 没有 self 概念，rB 只是字段所在对象）；argc 通过紧跟的 EXTRAARG 编码

### 闭包（2 条）

| OpCode | 格式 | 语法 | 语义 |
| --- | --- | --- | --- |
| CLOSURE | ABx | `CLOSURE rA, proto_idx` | 创建闭包，按 `protos[proto_idx].upvalue_descs` 装配 upvalue |
| CLOSE | ABC | `CLOSE rA` | 关闭 `rA` 及之后所有 open upvalue（作用域退出时） |

CLOSURE 装配过程：

```
for desc in proto.upvalue_descs:
  match desc:
    Local(reg):    在当前帧 open_upvalues 找已有 reg 对应 box；
                   不存在则新建并加入帧的 open_upvalues 列表；
                   闭包持有该 Rc 的克隆
    Upvalue(idx): 直接克隆当前闭包 upvalues[idx] 的 Rc
```

CLOSE 在每个块出口由 codegen 发出，将 OpenUpvalue 转为 ClosedUpvalue（值拷入 box）。

### Iterator 与协程（3 条）

| OpCode | 格式 | 语法 | 语义 |
| --- | --- | --- | --- |
| ITER | ABC | `ITER rA, rB` | `rA = iter(rB)`；非可迭代值则 throw |
| ITERLOOP | AsBx | `ITERLOOP rA, sBx` | 调用 `rA.next()`；若 done，跳 sBx；否则 `rA+1 = step.value` |
| YIELD | ABC | `YIELD rA` | 产出 `rA`，挂起 generator |

约定：

- ITERLOOP 隐含约定：`rA` 是 iterator 对象，`rA+1` 是循环变量
- `yield from` 在 codegen 阶段展开为 ITER + ITERLOOP + YIELD 循环，不需要专用指令
- coflow 不支持多值 yield，故 YIELD 不带 count

### 异常处理（3 条）

| OpCode | 格式 | 语法 | 语义 |
| --- | --- | --- | --- |
| TRY | AsBx | `TRY rA, sBx` | 注册 handler；handler PC = pc+sBx；err 寄存器 = rA |
| ENDTRY | ABC | `ENDTRY` | 弹出栈顶 handler |
| THROW | ABC | `THROW rA` | 抛 rA；unwind 栈到最近 handler；写入 err 寄存器；跳到 handler PC |

VM 维护 handler 栈，每个 handler 关联：

- handler PC
- 帧深度（栈 unwind 用）
- err 寄存器号

### 类型与运行时检查（2 条）

| OpCode | 格式 | 语法 | 语义 |
| --- | --- | --- | --- |
| TYPECHECK | ABx | `TYPECHECK rA, K_ty` | `rA` 类型与 `consts[K_ty]` 描述的 TypeDescriptor 不符则 throw |
| STACKCHECK | ABx | `STACKCHECK depth` | 检查当前调用栈深度未超过 `depth`，超过则 throw "调用栈过深" |

`TypeDescriptor` 在常量池中是一个序列化的 `Ty` 描述对象，能表达：

- 原语类型（Int / Float / Bool / String / Null）
- Class 引用（class_id）
- Enum 引用（enum_id）
- Array<T>
- Dict<K, V>
- FunctionSig（参数类型列表 + 返回类型）

VM 按 TypeDescriptor 递归校验值的运行时类型。

`STACKCHECK` 在每个函数入口和循环回边发出，防止递归爆栈。`depth` 通常是固定阈值（如 1000）。

## Inline Cache 设计（第二版加）

不增加指令数，但给 GETFIELD / SETFIELD / GETFIELDOPT / INVOKE / CALL 加 cache 槽位：

```rust
struct Proto {
    code: Vec<u32>,
    consts: Vec<Value>,
    consts_k: Vec<Value>,            // K 版本算术专用，最多 256 个
    field_names: Vec<String>,        // 最多 256 个
    upvalue_descs: Vec<UpvalueDesc>,
    locals_count: u16,
    line_info: Vec<(usize, Span)>,
    is_iter: bool,
    ic_slots: Vec<IcSlot>,           // 第二版加
}

struct IcSlot {
    class_id: u32,
    offset: u16,
}
```

ic_slot 通过指令 PC 索引（每条 GETFIELD / INVOKE 等指令对应一个 slot）。第一次执行时填充，命中时直接按 offset 取值。`INVOKE` 的 IC 同时缓存"字段偏移"和"目标函数"，热路径上避免每次都走字段查找。

## 常量池设计

主常量池 `consts: Vec<Value>` 16 位寻址（65536 上限）：

- 整数（超出 LOADINT 范围的）
- 浮点
- 字符串
- TypeDescriptor（用于 TYPECHECK）
- NamedArgList（用于 CALLKW）
- Proto 引用（用于 CLOSURE，子函数 Proto）

子常量池：

- `consts_k: Vec<Value>` 8 位寻址（256 上限）：算术 / 比较 K 版本的右操作数常量。codegen 把高频用到的常量数字也放在这里
- `field_names: Vec<String>` 8 位寻址（256 上限）：仅用于 GETFIELD / SETFIELD / GETFIELDOPT / INVOKE
- 单 Proto 超过 256 个不同字段名时编译报错（实际场景几乎不可能）

主常量池超过 65536 时，编译期建议拆分函数；如必须支持，第二版启用 LOADKX + EXTRAARG 兜底。

## 调用约定细节

调用 `f(a, b, c)`：

```
... 准备参数到连续寄存器 ...
MOVE rN, rF       # rN = 函数 f
MOVE rN+1, rA     # rN+1 = a
MOVE rN+2, rB     # rN+2 = b
MOVE rN+3, rC     # rN+3 = c
CALL rN, 3, 1     # argc=3, 期望 1 个返回值
... 此时 rN = 返回值 ...
```

具名参数 `f(a, x: b)`：

```
MOVE rN, rF
MOVE rN+1, rA
MOVE rN+2, rB
CALLKW rN, 2, K_namedlist
# K_namedlist 是 NamedArgList { entries: [(1, "x")] }
# 表示参数索引 1（即 rN+2）是具名参数 x
```

VM 处理 CALLKW：

1. 取 callee 的 Proto，知道形参列表
2. 把所有位置参数（去掉具名）按顺序填入形参
3. 把具名参数按名字填入对应形参
4. 缺省的形参填默认值
5. 不存在的形参名报"未知具名参数"
6. 重复填入同一形参报"参数冲突"

方法式调用 `weapon.apply(target)`：

```
MOVE rN+1, rTarget   # 参数先放好
INVOKE rN, rWeapon, K_apply
EXTRAARG 1           # argc = 1
```

VM 处理 INVOKE：

1. 通过 IC 或字段查找拿到 `rWeapon.<field_names[K_apply]>` 即函数对象
2. 写入 rN
3. 按 CALL 流程调用，参数从 rN+1 开始

## codegen 关键策略

### 寄存器分配

第一版采用 Lua 5.x 的虚拟寄存器栈方式：

- 局部变量按声明顺序占固定寄存器
- 表达式求值期间临时值按顺序分配寄存器栈顶
- 表达式结束后释放（栈顶下降）
- 不做完整线性扫描，简单且足够

### 跳转回填

发出跳转时填入 0xFFFF 占位，记录回填位置；目标确定后回写。循环回边发 JMP_LOOP 而非 JMP。

### TypeGuard 翻译

每个 `HirExpr::TypeGuard { expr, ty }` 翻成：

```
... gen(expr) → 结果 rA ...
TYPECHECK rA, K_ty
```

ty 序列化进常量池作为 TypeDescriptor。

### 链式 and 链翻译

`AndChain([e1, e2, e3])`：

```
gen(e1)  → rA
JFALSE rA, L_false
gen(e2)  → rA
JFALSE rA, L_false
gen(e3)  → rA
JMP L_end
L_false:
LOADBOOL rA, 0
L_end:
```

### if / while 翻译

if：

```
gen(cond)  → rA
JFALSE rA, L_else
gen(then_block)
JMP L_end
L_else:
gen(else_block)
L_end:
```

while：

```
L_start:
gen(cond)  → rA
JFALSE rA, L_end
gen(body)
JMP_LOOP L_start
L_end:
```

### null 检查翻译

`if x == null { ... }`：

```
BRNULL rA, L_then
... else 路径 ...
JMP L_end
L_then:
... null 分支 ...
L_end:
```

或简单直接：

```
ISNULL rT, rA
JTRUE rT, L_then
...
```

codegen 优先选 BRNULL 路径，省一个临时寄存器。

### ?. 翻译

`OptField` 直接发 GETFIELDOPT，VM 内部处理 null 短路。

### ?? 翻译

`NullCoalesce` 直接发 NULLCOALESCE。

### 数组字面量翻译

`[1, 2, 3, ..., 100]`：

```
NEWARRAY rA, 100
LOADINT rA+1, 1
LOADINT rA+2, 2
...
SETLIST rA, 100, 0
```

小数组（< 4 元素）退化为 NEWARRAY + APPEND 链。

### 字符串多元拼接

`a + b + c + d`：

```
... 准备 rA, rB, rC, rD 连续放置 ...
CONCATN rTarget, rA, rD
```

二元拼接走 CONCAT。

### 常量内联（K 版本）

HIR 中 BinaryExpr 的右操作数是 Const 时，codegen 优先发 K 版本：

- `x + 1` → `ADDK rA, rX, K_1`（K_1 是 consts_k 中数字 1 的索引）
- `x < 100` → `LTK 1, rX, K_100`

左操作数是常量时，对可交换运算（ADD/MUL/EQ）也可发 K 版本（交换操作数）。SUB/LT/LE 不可交换，左操作数是常量时仍走 LOADK + 通用版。

### 方法调用翻译

`obj.method(arg1, arg2)`：

```
... 准备 arg1, arg2 到 rA+1, rA+2 ...
INVOKE rA, rObj, K_method
EXTRAARG 2
```

### STACKCHECK 插入

每个 HirFunction 编译开始时发一条 STACKCHECK；每个循环 body 入口也发一条。

## 实现顺序

1. 定义 `OpCode` 枚举与 `Proto` 结构（`bytecode.rs`）
2. 实现指令编解码（packing/unpacking 32 位指令）
3. 实现常量池构建器（主池 + consts_k + field_names）
4. 实现 codegen 骨架（HIR → Proto），先支持表达式与基础语句
5. 实现寄存器分配（Lua 5.x 风格的虚拟栈）
6. 实现控制流 codegen（if / while / loop / for in / break / continue），用 JFALSE / JTRUE / JMP_LOOP
7. 实现闭包 codegen（CLOSURE / CLOSE / upvalue 链）
8. 实现 iter fn codegen（YIELD / ITERLOOP）
9. 实现 try/catch codegen
10. 实现 TypeGuard 插入与 TypeDescriptor 序列化
11. 实现 STACKCHECK 注入
12. 实现 CALLKW + NamedArgList
13. 实现 INVOKE（含 EXTRAARG）
14. 实现 SETLIST、CONCATN、LEN、ISNULL、BRNULL/BRNNULL、NULLCOALESCE
15. 实现 K 版本算术与比较（ADDK / SUBK / MULK / EQK / LTK / LEK）
16. 反汇编器（调试用）
17. 第二阶段：加分化指令、Inline Cache 槽、LOADKX 兜底
