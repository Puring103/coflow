# coflow 表达式

## 字面量

### 整数字面量

十进制整数：

```coflow
100
1_000_000
```

带进制前缀的整数：

```coflow
0xff         # 十六进制
0xFF         # 十六进制（字母大小写均可）
0b1010_0101  # 二进制
0o755        # 八进制
```

进制前缀的字母（`x` / `b` / `o`）必须**小写**。`0X10` / `0B10` / `0O10` 是词法错误。十六进制内部的数字字母（`a`–`f` / `A`–`F`）允许任意大小写。

数字分隔符 `_` 的规则：
- 只能出现在两个合法数字字符之间
- 不能出现在数字开头、结尾，不能连续出现（`__`）
- 带进制前缀时，前缀后至少需要一个合法数字

整数字面量的数值必须落在 `i64` 表示范围 `-2^63` ~ `2^63 - 1` 内（负号通过一元取负产生，词法层只匹配非负字面量，因此实际词法范围是 `0` ~ `2^63 - 1`）。超出范围的字面量是词法错误。例如 `0xFFFF_FFFF_FFFF_FFFF` 因超 `2^63 - 1` 非法；需要 64 位全 1 位模式时，使用 `~0`。

负号不是整数字面量的一部分，`-1` 解析为一元负号 `-` 加整数字面量 `1`。

### 浮点字面量

```coflow
3.5
1_000.000_5
1e3
1.0e-3
1E+3
```

规则：
- 小数点后必须紧跟数字或 `_`（`1.` 不合法，`.5` 不合法）
- 科学计数法：`e` 或 `E` 后可选 `+`/`-`，之后必须有至少一个数字
- `_` 只能出现在两个数字之间，不能出现在小数点两侧（`1_.0` 和 `1._0` 均非法）

### 布尔字面量

```coflow
true
false
```

### null 字面量

```coflow
null
```

### 字符串字面量

字符串字面量分为五种形态，由前缀和定界符区分：

| 形态 | 前缀/定界符 | 转义 | 插值 | 跨行 |
|------|-------------|------|------|------|
| 普通字符串 | `"..."` | 是 | 否 | 否 |
| 原始字符串 | `r"..."` | 否 | 否 | 否 |
| 多行字符串 | `"""..."""` | 是 | 否 | 是 |
| 原始多行字符串 | `r"""..."""` | 否 | 否 | 是 |
| 插值字符串 | `f"..."` 或 `f"""..."""` | 是 | 是 | 仅三引号形式 |

`f` 前缀和 `r` 前缀互斥（不存在 `rf"..."` / `fr"..."`）。原始字符串没有插值形式。

**普通字符串**：双引号包围，支持转义序列，不能跨行。

```coflow
"hello"
"line one\nline two"
"path: C:\\game"
```

支持的转义序列：

| 转义 | 含义 |
|------|------|
| `\"` | 双引号 `"` |
| `\\` | 反斜杠 `\` |
| `\n` | 换行符 LF |
| `\r` | 回车符 CR |
| `\t` | 水平制表符 |

普通字符串内出现未转义的换行符是词法错误。

**原始字符串**：以 `r"` 开始，以 `"` 结束，不处理转义序列，反斜杠视为普通字符，不能跨行。

```coflow
r"C:\game\assets\hero.png"
r"pattern: \d+"
```

**多行字符串**：以 `"""` 开始，以 `"""` 结束，保留内容中的换行，支持转义序列（与普通字符串相同）。

```coflow
var text = """
line one
line two
""";
```

**原始多行字符串**：以 `r"""` 开始，以 `"""` 结束，不处理转义序列。

```coflow
var shader = r"""
float4 main() {
  return float4(1, 0, 0, 1);
}
""";
```

**插值字符串**：以 `f"` 或 `f"""` 开始，定界规则与对应的非插值形式相同，转义规则与普通/多行字符串相同，并额外支持 `{ expr }` 内嵌表达式：

```coflow
var hp = 80;
var name = "hero";

var msg = f"hp: {hp}";                       # "hp: 80"
var greet = f"hello, {name}! ({hp} hp)";     # "hello, hero! (80 hp)"

var multi = f"""
name: {name}
hp:   {hp}
""";
```

规则：

- `{` 和 `}` 是插值定界符；要在插值字符串中输出字面 `{` / `}`，写 `{{` / `}}`
- `{ expr }` 内是任意表达式（**不**含语句、不含赋值、不含 `;`）。允许字段访问、调用、算术、`if` 表达式、`?.` 等
- 内嵌表达式不允许换行；表达式内部不允许包含未配对的 `{` 或 `}`
- 插值结果按 `string + 任意值` 的转换规则（见 [02-types.md](./02-types.md)）拼接为字符串
- 嵌套插值（`f"{ f\"...\" }"`）合法但不鼓励；嵌套层数和外层一致受词法解析限制

当所有 `{ expr }` 内嵌表达式都是常量表达式时，整个插值字符串本身也是常量表达式（参见 [07-config.md](./07-config.md)），可用于配置值和 `check` 块的诊断信息。

### 数组字面量

```coflow
[1, 2, 3]
["a", "b", "c"]
[]
```

元素之间用逗号分隔，允许末尾逗号。

### 对象字面量

对象使用 `{ }` 语法（**仅在表达式位置**，详见下文"花括号消歧"）。字段之间用逗号分隔，允许末尾逗号。

```coflow
{ id: "sword", damage: 10 }
```

对象字面量支持 `...` 展开：

```coflow
var base = { damage: 10, speed: 1.0 };
var sword = { ...base, name: "Iron Sword", damage: 15 };
```

展开可以出现在字段列表的任意位置，后面的同名字段覆盖前面的。

带类型名前缀的对象字面量 `TypeName{ ... }` 用于显式指定 class 类型（与配置定义中 `name: Type = { ... }` 等价但更紧凑）：

```coflow
var v = Vector{ x: 1.0, y: 2.0 };
```

### 字典字面量

字典使用 `dict{ }` 语法。条目之间用逗号分隔，允许末尾逗号。

```coflow
dict{ "alice": 10, "bob": 20 }
dict{}
```

对象与字典的区别见 [02-types.md](./02-types.md)。

### 花括号消歧

`{ ... }` 在 coflow 中可能是块、对象字面量或字典字面量。解析规则按 `{` 出现的位置分两类：**关键字驱动位置**（语法固定）和**自由位置**（表达式上下文）。

#### 1. 关键字驱动位置 — 永远是块

紧跟在以下关键字 / 语法形态之后的 `{` 永远开启块（语句序列）：

- `fn` / `iter fn`（函数体或 class 方法体）
- `class` / `enum`（声明体）
- `check`（校验块）
- `if` / `else`（分支体）
- `while` / `loop` / `for in`（循环体）
- `try` / `catch`（异常处理体）

这些位置不可能是对象字面量，因此不存在歧义：

```coflow
if cond {
  do_something();
}

while running { tick(); }

fn f() { return 1; }
```

#### 2. 自由位置 — 表达式上下文

非关键字驱动的位置（赋值右侧、参数实参、返回值、运算符两侧、`in` 右侧等），`{` 按下表判定：

| 形态 | 解析为 |
|------|--------|
| `dict{ ... }` | 字典字面量 |
| `Ident{ ... }`（标识符紧贴 `{`，无空白或操作符隔开） | 带类型名的对象字面量 |
| 其他表达式位置的裸 `{ ... }` | 对象字面量 |

空对象字面量写 `{}`，空字典写 `dict{}`。

```coflow
var a = { id: "x", damage: 10 };
var v = Vector{ x: 1.0, y: 2.0 };
var d = dict{ "k": 1 };
spawn({ name: "goblin" });
```

#### 3. 块表达式

coflow **不提供"块作为表达式"的语法形态**。需要"多语句 + 末尾值"的语义时，使用立即调用的匿名函数表达：

```coflow
var b = (fn() {
  var x = compute();
  return x * 2;
})();
```

简单的二选一计算优先使用 `if` 语句配合 `var`：

```coflow
var label;
if hp > 0 { label = "alive"; } else { label = "dead"; }
```

或使用 `if` 表达式（见下一节）——分支体形式与 `if` 语句一致。

## 名称

标识符引用当前作用域中的变量、函数或参数：

```coflow
hp
player
count
```

带 `.` 分隔的路径用于访问模块成员或枚举变体：

```coflow
Rarity.common
weapons.sword
game.utils.helper
```

## 函数表达式

匿名函数使用 `fn` 关键字，语法与顶层 `fn` 声明相同，但没有名称：

```coflow
fn(x) => x * 2

fn(name: string) -> string {
  return "hello " + name;
}

fn(a: int, b: int = 0) -> int => a + b
```

`iter fn` 匿名表达式：

```coflow
var gen = iter fn() {
  yield 1;
  yield 2;
};
```

## if 表达式

`if` 作为表达式时，分支体仍是关键字驱动位置的 `{ }` 块。块产生的值是**块内最后一条语句的值**：

- 若最后一条是表达式语句（`expr;`），值就是该表达式的值
- 若最后一条是不返回值的语句（赋值、`var` 声明、循环、`return` 等），整个块的值是 `null`

```coflow
var label = if hp > 0 { "alive"; } else { "dead"; };
var abs_val = if x >= 0 { x; } else { -x; };
```

要求：

- `if` 表达式的两个分支都必须存在
- 支持 `else if` 链：`if c1 { a; } else if c2 { b; } else { c; }`
- 表达式形式与语句形式语法相同；处于表达式位置时按表达式求值，处于语句位置时不取值
- 分支可以包含多条语句；只有最后一条语句的值参与表达式结果

```coflow
var score = if level >= 3 {
  log("hard mode");
  100 * level;
} else {
  10 * level;
};
```

注意 `;` 仍然必需（spec 的整体分号规则不放宽）。这是 coflow 与 Rust 的差异点：Rust 用"无分号"区分块的值表达式，coflow 一律要求分号，块的"值"是最后一条语句的值。

## 一元运算符

所有前缀一元运算符优先级相同，高于所有二元运算符：

| 运算符 | 含义 | 示例 |
|--------|------|------|
| `-`    | 取负 | `-hp` |
| `not`  | 逻辑非 | `not alive` |
| `~`    | 按位取反 | `~mask` |

## 二元运算符与优先级

下表按优先级从低到高排列（同一行优先级相同）：

| 优先级（低→高） | 运算符 | 结合性 | 说明 |
|----------------|--------|--------|------|
| 1  | `or`  | 左结合 | 逻辑或 |
| 2  | `and` | 左结合 | 逻辑与 |
| 3  | `??`  | 右结合 | 空值合并 |
| 4  | `\|`  | 左结合 | 按位或 |
| 5  | `^`   | 左结合 | 按位异或 |
| 6  | `&`   | 左结合 | 按位与 |
| 7  | `==` `!=` `<` `<=` `>` `>=` `in` `not in` | 链式 / 不可结合（详见下文） | 比较与成员判断 |
| 8  | `+` `-` | 左结合 | 加减 / 字符串拼接 |
| 9  | `<<` `>>` | 左结合 | 位移 |
| 10 | `*` `/` `//` `%` | 左结合 | 乘除取余 |
| 11 | `not` `~`（前缀） | — | 逻辑非 / 按位取反 |
| 12 | `**` | 右结合 | 幂运算 |
| 13 | `-`（前缀） | — | 一元取负 |
| 14 | `()` `.` `?.` `[]` `?[]`（后缀） | 左结合 | 调用与访问 |

注意 `**` 优先级**高于一元取负**：`-2 ** 2` 解析为 `-(2 ** 2) = -4`（与 Python 一致）。需要"先取负再幂运算"时显式加括号：`(-2) ** 2 = 4`。

### 逻辑运算

```coflow
if alive and not dead {
  update();
}

var name = input or "unknown";
```

`and` 和 `or` 短路求值：
- `a and b`：若 `a` 为假则返回 `a`，否则返回 `b`（不求值 `a` 为假时的 `b`）
- `a or b`：若 `a` 为真则返回 `a`，否则返回 `b`（不求值 `a` 为真时的 `b`）

### 空值合并

`a ?? b`：若 `a` 不为 `null` 则返回 `a`，否则返回 `b`。`??` 是右结合：

```coflow
var name = player?.name ?? config.default_name ?? "unknown";
# 等价于：player?.name ?? (config.default_name ?? "unknown")
```

### 比较运算

支持任意段数的**链式比较**：

```coflow
if 0 < damage <= 100 {
  apply(damage);
}
# 等价于：damage > 0 and damage <= 100

if a < b < c < d { ... }
# 等价于：a < b and b < c and c < d

if x == y == z { ... }
# 等价于：x == y and y == z
```

合法性规则：

1. 链中所有比较运算符的"方向"必须一致：
   - 全部为递增方向：`<` 与 `<=` 可以混用
   - 全部为递减方向：`>` 与 `>=` 可以混用
   - 全部为相等方向：仅 `==`
2. 不允许在同一链中混合不同方向（如 `a < b > c`）或将 `!=` 加入链（如 `a < b != c`）
3. `in` / `not in` 不参与链式比较；`a in b in c` 是非法

非法例子和诊断：

```coflow
a < b > c        # 错误：chained comparison must use consistent direction
a == b < c       # 错误：cannot mix equality and ordering in chained comparison
a < b != c       # 错误：'!=' cannot appear in chained comparison
```

求值语义：每个中间操作数只求值一次。`f() < g() < h()` 中 `g()` 只调用一次。任意一段为假即短路返回 `false`，剩余段不再求值。

链式比较外的比较运算（单段比较 `a < b`）的运算符**不可结合**，`(a < b) < c` 这种连续二元应用不允许——必须显式分开为多个语句或括号。

`in` / `not in` 永远是单段比较，不参与链式：`x in arr` 合法，`x in arr in matrix` 非法。

### 位运算

```coflow
var flags = STATUS_DEAD | STATUS_STUNNED;
if (flags & STATUS_DEAD) != 0 {
  die();
}
var cleared = flags & ~STATUS_STUNNED;
```

注意位运算符优先级高于比较运算符，但低于加减和乘除。在比较表达式中混用位运算通常需要加括号：

```coflow
if (flags & MASK) != 0 { ... }    # 正确
if flags & MASK != 0 { ... }      # 错误：解析为 flags & (MASK != 0)
```

### 整数除法与幂运算

```coflow
var q = damage // armor;   # 截断整除，结果为 int
var area = radius ** 2;    # 幂运算
```

`**` 右结合：`2 ** 3 ** 2` 等价于 `2 ** (3 ** 2)` = 512。

### 成员判断

`in` 和 `not in` 判断元素是否属于集合。`not in` 是**原子词法 token**，必须连写，且其间只允许一个空格分隔；写成 `not (x in y)` 不等价（前者是单一比较运算 `not in`，后者是逻辑非作用于一个比较结果，运算优先级不同）。

```coflow
if item in items { ... }        # 数组成员判断
if key in scores { ... }        # 字典 key 判断
if hp in 1..=100 { ... }        # Range 范围判断
if status not in banned { ... } # 非成员判断
```

`not foo in bar` 是语法错误：`not` 不能直接修饰 `in` 表达式。需要表达"不属于"时使用 `not in`；需要表达"对成员判断结果取反"时显式加括号 `not (foo in bar)`（虽然语义同 `not in`，但写法上要明示）。

## 后缀操作

后缀操作的优先级最高，从左到右依次应用。

### 函数调用

```coflow
fn_name(arg1, arg2)
obj.method(arg1)
```

**具名参数**：调用时以 `param_name: value` 形式传入具名参数：

```coflow
spawn("goblin");
spawn("boss", hp: 500);
spawn("elite", 200, 1);
```

规则：具名参数必须与形参名一致；位置参数和具名参数可以混用；位置参数之后可以接具名参数，但具名参数之后不能再有位置参数。

### 字段访问

```coflow
obj.field
player.profile.name
```

普通字段访问要求字段必须存在。字段不存在时报运行时错误。

### 可选字段访问

```coflow
obj?.field
```

若 `obj` 为 `null` 或字段不存在，结果为 `null`，不抛错。可以链式使用：

```coflow
var name = player?.profile?.name ?? "unknown";
```

若 `obj` 的静态类型是封闭 class，访问 class 未声明字段是语义错误，`?.` 不能绕过静态结构校验。

### 索引访问

```coflow
arr[0]
dict["key"]
```

普通索引访问是严格访问。数组越界或字典 key 不存在时报运行时错误。

### 可选索引访问

```coflow
arr?[0]
dict?["key"]
```

若左侧为 `null`、数组越界或字典 key 不存在，结果为 `null`，不抛错。

## Range 表达式

Range 产生可迭代的整数序列，只能用于 `for in` 或 `in` 判断：

```coflow
0..10    # [0, 10)，不含 10
0..=10   # [0, 10]，含 10
```

```coflow
for i in 0..10 { ... }

if hp in 1..=100 { ... }
```

Range 的起始和终止都是整数表达式：

```coflow
for i in start..(start + count) { ... }
```

## 括号分组

圆括号 `( )` 用于显式控制求值顺序：

```coflow
var result = (a + b) * c;
if (flags & MASK) != 0 { ... }
```
