# coflow 语言概览

coflow 是一门面向嵌入式场景的动态脚本语言，目标是替代 Lua，并把配置作为语言的一等能力。

## 设计目标

1. 轻量，可嵌入。
2. 适合作为宿主程序的脚本语言。
3. 配置和脚本使用同一种语法。
4. 顶层配置由宿主统一加载、校验和消费。
5. 动态语言为底，类型标注服务于配置校验和宿主互操作。
6. 控制流和数据模型清晰，避免 Lua table 式语义过载。

## 源码编码

coflow 源文件使用 UTF-8 编码。

## 空白与换行

空白字符包括空格（`U+0020`）、制表符（`U+0009`）、换行（`U+000A`）、回车（`U+000D`）和换页（`U+000C`）。空白仅用于分隔语法单元，无语义意义。

coflow 是自由格式语言，换行不作为语句终止符。

## 分号

分号 `;` 是**必须的**语句终止符，用于结束以下构造：

- `import` 声明
- `var` 声明（顶层和块内）
- 配置定义（`name = value`）
- 赋值语句和复合赋值语句
- 表达式语句（函数调用等）
- `break`、`continue`、`return`、`throw`
- `yield`、`yield from`
- `class` 内的字段、方法、`check` 块之间的分隔
- `enum` 内的变体之间的分隔
- `check` 块内的 `assert` 语句

以 `}` 结尾的构造**不需要**分号，包括：`fn`、`iter fn`、`class`、`enum` 声明，以及 `if`、`while`、`loop`、`for in`、`try catch` 语句。

```coflow
import weapons;             # import 需要分号

var hp = 100;               # var 需要分号
hp -= 10;                   # 赋值需要分号
print(hp);                  # 表达式语句需要分号

fn take_damage(amount) {    # fn 声明不需要分号
  hp -= amount;             # 块内赋值需要分号
  return hp;                # return 需要分号
}

if hp <= 0 {                # if 不需要分号
  die();                    # 调用表达式需要分号
}
```

注意：当 `return`、赋值等语句的值是以 `}` 结尾的表达式（如匿名函数、对象字面量、块表达式）时，分号加在 `}` 之后：

```coflow
fn make_counter() {
  var count = 0;
  return fn() {
    count += 1;
    return count;
  };             # return 语句的分号在匿名函数的 } 之后
}
```

## 注释

### 行注释

以 `#` 开头，直到行末。

```coflow
# 这是一行注释
var hp = 100;  # 行尾注释
```

### 块注释

以 `/*` 开始，以 `*/` 结束，可以跨越多行。

```coflow
/*
  多行注释
  可以跨越多行
*/
var hp = 100;
```

块注释不支持嵌套。`/*` 之后遇到的第一个 `*/` 即结束注释。

## 标识符

标识符由 Unicode XID 字符集定义：

- 首字符：满足 `XID_Start` 的 Unicode 字符，或 `_`（下划线）。
- 后续字符：满足 `XID_Continue` 的 Unicode 字符，或 `_`。

实践中，ASCII 字母、数字、下划线构成最常见的标识符，也支持中文等非 ASCII 字母字符。

```coflow
var hp = 100;
var _private = 0;
var 生命值 = 100;
```

标识符区分大小写。`hp`、`HP`、`Hp` 是三个不同的标识符。

## 关键字

以下标识符是保留关键字，不能用作普通标识符：

```
and       as        assert    break     catch
check     class     continue  dict      else
enum      false     fn        for       from
if        import    in        iter      local
loop      not       null      or        return
self      throw     true      try       var
while     yield
```

`env` 是加载期注入的全局常量命名空间，不是关键字，但作为预定义全局名不可被重新声明或遮蔽。详见 [07-config.md](./07-config.md)。

## 运算符与分隔符

### 算术运算符

| 运算符 | 含义 |
|--------|------|
| `+`    | 加法 / 字符串拼接 |
| `-`    | 减法 / 一元取负 |
| `*`    | 乘法 |
| `/`    | 除法 |
| `//`   | 整数除法（截断） |
| `%`    | 取余 |
| `**`   | 幂运算（右结合） |

### 比较运算符

| 运算符 | 含义 |
|--------|------|
| `==`   | 等于 |
| `!=`   | 不等于 |
| `<`    | 小于 |
| `<=`   | 小于等于 |
| `>`    | 大于 |
| `>=`   | 大于等于 |

### 逻辑运算符

| 运算符 | 含义 |
|--------|------|
| `and`  | 逻辑与 |
| `or`   | 逻辑或 |
| `not`  | 逻辑非（前缀一元） |

### 位运算符

| 运算符 | 含义 |
|--------|------|
| `&`    | 按位与 |
| `\|`   | 按位或 |
| `^`    | 按位异或 |
| `~`    | 按位取反（前缀一元） |
| `<<`   | 左移 |
| `>>`   | 右移 |

### 其他运算符

| 运算符   | 含义 |
|----------|------|
| `??`     | 空值合并 |
| `?.`     | 可选字段访问 |
| `?[]`    | 可选索引访问 |
| `..`     | Range（不含尾） |
| `..=`    | Range（含尾） |
| `in`     | 成员判断 |
| `not in` | 非成员判断 |
| `...`    | 展开（对象字面量） |

### 赋值运算符

| 运算符  | 含义 |
|---------|------|
| `=`     | 赋值 |
| `+=`    | 加法赋值 |
| `-=`    | 减法赋值 |
| `*=`    | 乘法赋值 |
| `/=`    | 除法赋值 |
| `??=`   | 空值合并赋值 |

复合赋值仅保留以上五种。位运算、整除、取余、幂运算的复合赋值在配置/嵌入式场景使用频次极低，统一展开为完整赋值（`x = x // y` 等）。

### 分隔符

| 符号  | 用途 |
|-------|------|
| `(`   | 函数调用、参数列表、分组表达式 |
| `)`   | 同上（结束） |
| `[`   | 数组字面量、索引访问 |
| `]`   | 同上（结束） |
| `{`   | 块、对象字面量、`dict` 字典字面量；详见 [03-expressions.md](./03-expressions.md) 的"花括号消歧" |
| `}`   | 同上（结束） |
| `,`   | 分隔参数、字段、元素 |
| `:`   | 类型标注、对象字段键值分隔 |
| `->`  | 返回类型标注 |
| `=>`  | 函数体简写（fat arrow） |
| `.`   | 字段访问、路径分隔 |
| `;`   | 语句终止符 |
