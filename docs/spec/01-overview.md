# coflow 语言概览

coflow 是一套面向嵌入式场景的动态语言系统，由可执行脚本文件 `.cfs` 和对象图数据文件 `.cfc` 组成。

`.cfs` 用于编写宿主程序中的运行时逻辑、工具脚本和自动化流程。它是轻量的动态脚本语言，适合嵌入宿主程序，目标是在保持 Lua 式易嵌入性的同时，提供更清晰的数据模型、模块边界和宿主互操作能力。

`.cfc` 用于保存结构化数据。它是纯数据文件，不执行代码。与 JSON 这类树形数据格式不同，`.cfc` 保存的是对象图：命名数据节点具有稳定 identity，多个位置引用同一节点时，加载后仍保持共享引用。`.cfc` 因此既适合人工编辑和工具编辑，也适合保存需要保留引用关系的复杂数据。

coflow 的核心特色是把脚本执行和对象图持久化作为同等重要的基础能力。脚本可以加载 `.cfc` 数据、操作对象图，并在宿主授权下写回 `.cfc`。数据文件保持无副作用和可分析性，脚本文件承担运行时逻辑和宿主交互。

## 设计目标

1. 轻量，可嵌入，适合作为宿主程序的脚本语言。
2. 提供清晰的数据模型，区分对象、数组和字典，避免单一 table 承担过多语义。
3. 提供语言级对象图持久化能力，保存和恢复共享引用关系。
4. 数据文件保持纯数据语义，便于外部工具读取、编辑、校验和生成。
5. 脚本文件提供直接的运行时表达能力，便于调用宿主 API、编写工具逻辑和操作数据。
6. 动态语言为底，类型和 schema 服务于数据校验、宿主互操作和编辑器支持。
7. 宿主程序控制加载、保存、权限、运行时 API 和生命周期。

## 文件类型

coflow 使用两类源文件：

- `.cfs`：脚本文件。用于执行逻辑，支持模块、函数、变量、控制流和宿主 API。
- `.cfc`：对象图数据文件。用于保存纯数据，支持结构化对象、数组、字典、schema、枚举、命名节点和模块化数据引用。

`.cfs` 面向执行，`.cfc` 面向持久化。二者共享基础值模型，使脚本可以自然地读取、修改和保存数据对象。

## 核心价值

Lua 的优势在于轻量和易嵌入，但 table 同时承担对象、数组、字典、模块和实例等多种角色，复杂项目中容易产生语义过载。

JSON 的优势在于工具友好和数据安全，但只能表达树形数据，缺少类型、模块化和对象 identity。

coflow 试图结合两者的关键优势：像 Lua 一样适合嵌入和执行，像 JSON 一样适合工具处理和数据交换，同时提供对象图持久化、schema 校验和更明确的数据结构。
