Coflow / Game Config Pipeline

项目介绍以及想解决的问题

Coflow 是一个面向游戏配置数据的 Rust 工具链，把 schema、Excel、文本配置、 数据校验、导出格式和 C# 运行时代码放进同一条可重复执行的流程。

配置语义分散

类型定义、表格列、运行时代码和校验规则分散维护时，很容易出现字段漂移。

错误定位困难

数据错误如果只在运行期暴露，策划和开发都难以快速定位到文件、sheet、行列或字段。

交付链路脆弱

手写导出脚本和加载器容易遗漏默认值、引用、多态、nullable、枚举和字典边界。

项目如何解决这些问题

Coflow 用 CFT 作为配置语义的事实来源，所有数据源和产物都围绕编译后的 schema 工作。

用 schema 统一语义

CFT 定义类型、继承、多态、enum、默认值、nullable、引用和校验规则。

用 loader 收敛输入

Excel 和 CFD 都转换成统一输入记录，后续不关心数据来自表格还是文本。

用模型校验不变量

data model 负责字段类型、必填字段、重复 key、引用目标和路径引用。

用产物保持一致

JSON、MessagePack 和 C# codegen 都从同一份 schema/model 推导。

项目的核心架构

架构按职责分层：底层只表达语言和模型语义，上层只做数据源适配、管线编排和工具集成。

Project读取 coflow.yaml，发现 schema 和 sources。
CFT编译 schema，产生类型系统和诊断。
Loader读取 Excel / CFD，输出输入记录。
Cell Value按字段类型解析单元格复合值。
Data Model构建表、记录、索引、默认值和引用。
Checker执行 CFT check，返回数据诊断。
Output导出数据并生成 C# 运行时代码。

语言与模型层

`coflow-cft`、`coflow-api::cell_value`、`coflow-cfd`、`coflow-data-model` 负责核心语义。

管线与诊断层

`coflow-project`、`coflow-loader-excel`、`coflow-checker`、`coflow-pipeline` 负责编排和定位。

产物与工具层

`coflow-exporter-*`、`coflow-codegen-csharp`、CLI 和 LSP 对外提供可用入口。

简单的示例

一个最小项目通常包含 schema、数据源和输出配置。下面的示例展示从字段定义到校验、导出和代码生成的基本路径。

1. 定义 schema

用 CFT 描述配置表结构和校验规则。

enum Rarity { Common, Rare, }

type Item {
  name: string;
  rarity: Rarity = Rarity.Common;
  price: int;

  check {
    name != "";
    price >= 0;
  }
}

2. 配置项目并运行

Excel sheet 的 `id` 列作为 record key，其余列按字段名或 columns 映射读取。

schema: schema/
sources:
  - file: data/items.xlsx
    sheets:
      - sheet: Item
outputs:
  data:
    type: json
    dir: generated/data
  code:
    type: csharp
    dir: generated/csharp
    namespace: Game.Config
cargo run -- check examples/rpg
cargo run -- build examples/rpg
