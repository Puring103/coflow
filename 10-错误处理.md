# coflow错误处理设计

coflow第一版采用极简异常式错误处理
错误处理主要面向运行时错误和宿主API错误

# 错误类型
错误对象至少包含message和stack

```coflow
class Error {
  message: string
  stack: string
}
```

# throw
使用throw抛出错误

```coflow
fn check_hp(hp: int) {
  if hp < 0 {
    throw "hp must be >= 0"
  }
}
```

throw string时运行时自动包装为Error

# try catch
使用try catch捕获运行时错误

```coflow
try {
  risky()
} catch err {
  print(err.message)
}
```

catch绑定一个错误变量

# 宿主错误
宿主API错误可以映射为coflow Error

```coflow
try {
  file.read("missing.txt")
} catch err {
  print(err.message)
}
```

# 协程错误
协程内部发生错误时
resume传播运行时错误
协程进入dead状态

```coflow
co fn task() {
  throw "failed"
}

var c = task()

try {
  c.resume()
} catch err {
  print(err.message)
}
```

# 配置错误
配置错误不通过try catch捕获
配置错误属于编译期或加载期诊断

```coflow
bad: Weapon = {
  damage: "high"
}
```

这类错误应作为配置校验错误报告
不是脚本可捕获异常

# 暂不支持
1. finally
2. 多catch
3. catch类型匹配
4. try表达式
5. checked exception
6. Result类型语法糖
